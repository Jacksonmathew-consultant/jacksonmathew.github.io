    Google Search Experience Enhancement

    A proposal to make search more accessible, accurate, and reliable for all users.

    Project Snapshot

    Project Type: Product Proposal

    My Role: Product Strategist

    Tools & Frameworks

    User Personas

    RICE Scoring

    AI/Bard Integration

    Accessibility (A11y) Principles

    Problem & Goal

    While Google Search is the world's leading search engine, it faces challenges with biased or inaccurate results, accessibility for users with disabilities, and the spread of misinformation. The goal was to develop a strategic proposal to address these issues through targeted features for distinct user personas (visually impaired users, technical users, and students).

    Role & Responsibilities

    As a Product Strategist, I was responsible for the entire proposal. This included conducting market research, defining user personas and their pain points, ideating solutions, creating mockups to visualize the features, and using the RICE framework to prioritize the proposed initiatives based on Reach, Impact, Confidence, and Effort.

    Outcomes (Metrics)

    The North Star Metric for this proposal was an increase in user retention and satisfaction. Key success metrics included:

    • Increase in daily active users for the voice-only search mode.
    • Reduction in bounce rate for technical search queries.
    • Higher user satisfaction scores (NPS) from student users.
    • Increase in usage of fact-checked annotations and feedback buttons.

    Visuals

    The proposal was supported by user journey maps, wireframes for the new search result features, and a prioritization matrix based on the RICE score.

    Myntra Onboarding Experience Redesign

    Enhancing the new user journey for India's largest fashion e-commerce platform.

    Project Snapshot

    Project Type: UX Analysis & Proposal

    My Role: Product Analyst

    Tools & Frameworks

    Figma

    User Journey Mapping

    Pain Point Analysis

    Gamification

    Problem & Goal

    Despite a large user base, Myntra's new user onboarding process had a high bounce rate (52.88%). New users were faced with overwhelming choice, complex navigation, and a lack of personalization. The goal was to redesign the onboarding flow to reduce friction, increase engagement, and guide new users toward key actions, ultimately improving conversion and retention for this critical user segment.

    Role & Responsibilities

    As a Product Analyst, I was responsible for analyzing the existing user journey, identifying key pain points, and proposing a data-informed solution. I created detailed user flows, designed wireframes and mockups in Figma for the new onboarding process, and developed concepts for new features like personalization quizzes and a virtual try-on tool. The final output was a comprehensive proposal outlining the problems, solutions, and expected impact.

    Outcomes (Metrics)

    The proposal aimed to directly impact key business metrics:

    • Decrease Bounce Rate: Reduce the initial bounce rate for new users by 15-20%.
    • Increase Conversion Rate: Improve the first-purchase conversion rate for new users by 10%.
    • Boost Engagement: Increase the number of items added to the wishlist and the average session duration for new users.
    • Improve Retention: Decrease churn rate for users within the first 30 days.

    Visuals

    The proposal included detailed user journey maps (current vs. proposed), wireframes for the new personalization screens, and high-fidelity mockups of the virtual try-on feature.
